⚙️ Plan de veille (SLAM)

Le suivi continu permet de ne pas stagner et d'anticiper l'évolution des infrastructures modernes, alliant la puissance historique du noyau Linux à l'agilité des assistants de code actifs.

📄 Articles & Axes d'Étude
Sujet 1 : Les assistants de code IA locaux sous environnement Linux
Stack : Ollama · VS Code · DeepSeek-Coder
Synthèse
Étude de l'intégration des modèles de langage (LLM) exécutés localement sur des machines Linux pour assister le développement. L'implémentation d'outils comme Ollama permet aux développeurs SLAM de bénéficier d'une complétion de code intelligente et d'une génération algorithmique performante sans dépendre d'API cloud tierces, garantissant ainsi la confidentialité absolue du code source de l'organisation.
Sujet 2 : Automatisation des scripts système et DevOps par l'IA
Stack : Bash · Python · ShellGPT
Synthèse
Analyse de la convergence entre l'administration Linux et le développement à travers l'utilisation d'outils CLI propulsés par l'IA (comme ShellGPT). Ce flux de travail permet de générer des scripts Bash complexes, d'automatiser le déploiement de conteneurs Docker, et de diagnostiquer instantanément les fichiers de logs système grâce au traitement du langage naturel, optimisant ainsi le temps de maintenance applicative.

Documentation Technique — Okantis

Dans le cadre de mon BTS SIO, j'ai réalisé la modernisation et la refonte d'outils d'extraction pour la suite hospitalière M-RH. J'ai migré des scripts Shell Linux historiques vers des packages PL/SQL Oracle natifs (PCK_ANU_BRIVE) afin d'optimiser la sécurité, la performance (via UTL_FILE) et la maintenance. J'ai ensuite fait évoluer la solution vers une architecture générique multi-établissements (PCK_ANU_GENERIQUE).
